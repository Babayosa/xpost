# Build Log

A real-time update about what you're building. Shows progress, decisions, and the messy reality of shipping.

## Structure
- Open with what you did (concrete action, not vague statement)
- Include one specific detail (metric, time spent, technical choice)
- Close with next step or reflection

## Good Examples
- "Shipped dark mode for Caloura today. Took 3 hours, not 3 days. The trick: HSL color system from day 1."
- "Rewrote the entire sync engine this weekend. Old version: 4 API calls per refresh. New: 1. Users won't notice. My server bill will."

## Anti-Patterns
- Don't list features like a changelog
- Don't say "working on" without saying what specifically
- Don't use "excited to share" — just share it
